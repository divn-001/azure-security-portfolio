# Cloud Posture Management

## Purpose
This section demonstrates how cloud security posture is established and monitored
using Microsoft Defender for Cloud.

The focus is on visibility, risk prioritization, and baseline hardening.

---

## Scope (Strictly Enforced)

### In Scope
- Defender for Cloud
- Secure Score analysis
- Baseline security recommendations
- Regulatory compliance views

### Explicitly Out of Scope
- Governance enforcement (see `04-governance-and-landing-zones`)
- Threat detection and incident response

---

## Capabilities Demonstrated

### 1. Establish Security Baselines
- Identify high-impact misconfigurations
- Prioritize remediation efforts
- Understand cloud risk exposure

### 2. Monitor and Improve Posture
- Use Secure Score meaningfully
- Track improvement over time
- Align recommendations to business risk

---

## Execution Philosophy
Posture management is used to **inform decisions**, not blindly enforce controls.
Recommendations are evaluated before adoption.

---

## Success Criteria
- Security posture risks are clearly understood
- Improvements are intentional and measurable
- Baselines can be justified and defended
